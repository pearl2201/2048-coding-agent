# Game Design Document: 2048

## 1. Project Overview

**Project Name:** 2048  
**Type:** Puzzle / Number-merging casual game  
**Core Functionality:** A sliding tile puzzle where players combine numbered tiles to reach the target score of 2048.  
**Target Users:** Casual gamers of all ages seeking a quick, addictive brain teaser.

---

## 2. Visual & Rendering Specification

### Scene Setup
- **Layout:** 4x4 grid centered on screen with rounded tile cells
- **Background:** Soft gradient background (muted warm tones – e.g., cream to light beige)
- **Grid:** Visible cell outlines with subtle borders (rounded corners)

### Materials & Effects
- **Tiles:** Flat-colored tiles with numbers displayed in bold sans-serif font
- **Color Palette:** Each number has a distinct color (progression from light to dark tones):
  - 2 → #eee4da
  - 4 → #ede0c8
  - 8 → #f2b179 (orange tint)
  - 16 → #f59563
  - 32 → #f67c5f
  - 64 → #f65e3b
  - 128 → #edcf72 (yellow)
  - 256 → #edcc61
  - 512 → #edc850
  - 1024 → #edc53f
  - 2048 → #edc22e
- **Text:** Dark text (#776e65) on light tiles, light text (#f9f6f2) on darker tiles

### Animations
- **Tile Slide:** Smooth CSS transitions (~150ms ease-out)
- **Tile Merge:** Brief scale pulse animation on merge
- **New Tile:** Fade-in + scale-up animation when new tile spawns
- **Win/Lose Overlay:** Semi-transparent overlay with modal popup

---

## 3. Game Mechanics Specification

### Core Loop
1. Player presses arrow key (or swipes on mobile)
2. All tiles slide in the chosen direction
3. Adjacent tiles with the same value merge into one (doubled value)
4. A new tile (90% chance of "2", 10% chance of "4") spawns in a random empty cell
5. Score increases by the value of merged tiles
6. Repeat until win condition (reached 2048) or lose condition (no moves left)

### Rules
- **Merging:** Only one merge per tile per move; a tile created from a merge will not merge again until the next move
- **Scoring:** Each merge adds the resulting tile value to the score
- **Win Condition:** Creating a tile with value 2048 (can continue playing)
- **Lose Condition:** No empty cells AND no adjacent tiles can merge in any direction

### Controls
- **Desktop:** Arrow keys (↑↓←→) to slide tiles
- **Mobile:** Swipe gestures (up/down/left/right)
- **New Game Button:** Reset the board and score

---

## 4. UI Specification

### Header
- Game title "2048" (bold, large font)
- Current score display
- Best score display (stored in localStorage)

### Game Board
- 4x4 grid of tiles
- Each cell either empty or containing a numbered tile

### Footer
- "New Game" button
- Instructions text: "Use arrow keys to move tiles"

### Win/Lose Overlay
- **Win:** "You Win!" message with options: "Keep Playing" or "New Game"
- **Lose:** "Game Over!" message with "Try Again" button

---

## 5. Technical Implementation

### Technology
- HTML5 + CSS3 + Vanilla JavaScript
- No external libraries required (pure implementation)
- Responsive design for desktop and mobile

### Data Structures
- **Board State:** 4x4 2D array of integers (0 = empty)
- **Score:** Integer tracking current score
- **Best Score:** Integer persisted in localStorage

### Key Functions
- `initGame()` – Initialize board with two random tiles
- `move(direction)` – Handle tile movement and merging
- `mergeTiles()` – Process tile merges and update score
- `spawnTile()` – Add new tile to random empty cell
- `checkWin()` – Check if 2048 tile exists
- `checkGameOver()` – Check if any moves are possible
- `updateDisplay()` – Render current board state to UI

## 6. Acceptance Criteria

1. ✓ Game initializes with a 4x4 grid containing two random tiles (2 or 4)
2. ✓ Arrow keys slide all tiles in the pressed direction
3. ✓ Adjacent tiles with matching values merge into one (doubled value)
4. ✓ Score updates correctly on each merge
5. ✓ New tile spawns after each valid move
6. ✓ Game detects win condition (2048 tile created)
7. ✓ Game detects lose condition (no valid moves remaining)
8. ✓ New Game button resets the board and score
9. ✓ Best score persists across browser sessions (localStorage)
10. ✓ Responsive design works on both desktop and mobile browsers
11. ✓ Smooth animations for tile movement, merging, and spawning
12. ✓ Color scheme matches specification for all tile values
