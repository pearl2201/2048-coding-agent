## 1. Executive Summary

### 1.1 Elevator Pitch
*2048* is a single-player, sliding-tile puzzle game played on a 4×4 grid. The player swipes to combine numbered tiles; identical numbers merge into their sum. The goal is to create the 2048 tile — and then keep going for a high score.

### 1.2 Target Audience
- **Primary:** Casual gamers, puzzle enthusiasts, mobile users ages 12–55.
- **Secondary:** Competitive score-chasers who share results on social media.

### 1.3 Platform & Tech Stack
- **Platforms:** Web (HTML5/CSS/JS), iOS, Android.
- **Tech:** React (or vanilla JS), Canvas/DOM rendering, touch + keyboard input.

## 2. Gameplay & Mechanics

### 2.1 Core Loop

```
┌─────────────────────────────────────┐
│  1. Observe the 4×4 grid           │
│  2. Swipe/arrow in one direction   │
│  3. All tiles slide as far as      │
│     possible in that direction     │
│  4. Adjacent identical tiles merge │
│  5. A new tile (2 or 4) spawns     │
│  6. Check win/lose conditions      │
│  7. Repeat                         │
└─────────────────────────────────────┘
```

### 2.2 Rules (Detailed)

| Rule | Description |
|------|-------------|
| **Grid** | 4×4 cells (16 positions). Each cell holds one tile or is empty. |
| **Slide** | All tiles move as far as possible in the chosen direction (up, down, left, right). Tiles stop at the edge or against another tile that cannot merge. |
| **Merge** | When two tiles with the same number collide during a slide, they merge into **one** tile with value `N × 2`. Each tile can merge **only once per move**. |
| **Spawn** | After every valid move, a new tile (90% chance of `2`, 10% chance of `4`) appears in a random empty cell. |
| **Scoring** | Each merge adds the merged tile's value to the score. E.g., merging two `4`s adds `+8`. |
| **Win** | Reaching the `2048` tile triggers a win. The player may continue playing ("Keep Going"). |
| **Lose** | No empty cells AND no adjacent equal tiles = Game Over. |

### 2.3 Move Resolution Algorithm

1. For the chosen direction, iterate rows/columns in the order tiles will move.
2. For each line (row or column):
   - **Compress:** Remove all empty spaces (shift non-zero tiles toward the movement edge).
   - **Merge:** Scan from the movement edge inward; if two adjacent tiles are equal and the first hasn't already merged this turn, combine them. Place the merged tile closer to the edge and leave a gap.
   - **Compress again** to fill gaps left by merges.
3. If the grid changed, spawn a new tile and increment score.

| Step | Value | Tile Color |
|------|-------|------------|
| 1 | 2 | #EEE4DA |
| 2 | 4 | #EDE0C8 |
| 3 | 8 | #F2B179 |
| 4 | 16 | #F59563 |

## 3. User Interface & Controls

### 3.1 Controls

| Input | Action |
|-------|--------|
| Arrow keys (↑ ↓ ← →) | Slide tiles |
| W/A/S/D | Slide tiles |
| Swipe gesture | Slide tiles (mobile) |
| "New Game" button | Reset board |
| "Undo" button (optional) | Revert last move (limited uses) |

### 3.2 HUD Layout (Mobile)

```
┌──────────────────────────┐
│  2048          Best: 4096│
│  Score: 1248             │
├──────────────────────────┤
│                          │
│     4×4 Game Grid       │
│                          │
├──────────────────────────┤
│  [New Game]  [Undo]      │
└──────────────────────────┘
```

### 3.3 Animations
- **Slide:** Tween tiles smoothly (100–150ms) to new positions.
- **Merge:** Scale pop (1.0 → 1.2 → 1.0) over 150ms.
- **Spawn:** Fade-in / scale-up (0.0 → 1.0) over 120ms.
- **Win overlay:** Semi-transparent backdrop with "You Win!" and "Keep Going" button.

## 4. Game States & Transitions

```
        ┌──────────┐
        │  IDLE    │ ← game loaded, waiting for first move
        └────┬─────┘
             │ any swipe/key
        ┌────▼─────┐
   ┌───►│ PLAYING  │◄─── "Keep Going"
   │    └────┬─────┘
   │         │
   │    ┌────▼─────┐     ┌─────────┐
   │    │  WIN     │────►│ VICTORY │ (player chooses "Keep Going" → PLAYING)
   │    └────┬─────┘     └─────────┘
   │         │
   │    ┌────▼─────┐
   └────│  LOSE    │
        └──────────┘
```

### 4.1 State Descriptions

| State | Description |
|-------|-------------|
| **IDLE** | Grid populated with 2 starting tiles. Waiting for input. |
| **PLAYING** | Active game loop. Accepts moves. |
| **WIN** | 2048 tile achieved. Show congratulations. Offer "Keep Going" to continue past 2048. |
| **LOSE** | No valid moves remain. Show "Game Over" and final score. Offer "New Game." |
| **PAUSED** (optional) | Game suspended; grid hidden or blurred. |

## 5. Scoring & Persistence

### 5.1 Scoring Formula
```
score += merged_tile_value
```
No bonus multipliers. Simple and transparent.

### 5.2 Best Score Persistence
- **Local:** `localStorage` key `2048_best_score`.
- **Online (future):** Leaderboard via backend API.

### 5.3 Game State Persistence
- Save the full grid state, current score, and best score to `localStorage` after every move.
- On app load, restore saved state if present; otherwise start fresh.
- Provide "Reset Game" / "New Game" to clear state.

## 7. Technical Architecture

### 7.1 Data Model

```typescript
interface GameState {
  grid: number[][];          // 4×4, 0 = empty
  score: number;
  bestScore: number;
  status: 'idle' | 'playing' | 'win' | 'lose';
  moveHistory: Move[];       // For undo
}

interface Move {
  gridBefore: number[][];
  scoreBefore: number;
}
```

### 7.2 Key Functions

| Function | Purpose |
|----------|---------|
| `initGame()` | Create empty grid, spawn 2 starting tiles, set status to `idle`. |
| `slide(direction)` | Execute the slide-and-merge logic for the given direction. |
| `spawnTile()` | Place a random `2` or `4` in an empty cell. |
| `checkWin()` | Returns true if any tile is ≥ 2048. |
| `checkLose()` | Returns true if no moves are possible. |
| `saveState()` / `loadState()` | Persist/restore from `localStorage`. |

### 7.3 Performance Considerations
- Grid is only 4×4; no significant performance concerns.
- Animation should use CSS transitions or `requestAnimationFrame`.
- Ensure move resolution is synchronous and fast (< 1ms).

## 8. Edge Cases & Error Handling

| Scenario | Handling |
|----------|----------|
| No valid move in given direction | Grid unchanged; no new tile spawned; no turn consumed. |
| Multiple merges in one line | Only the first eligible pair (from the movement edge) merges. E.g., `[4,4,4,0]` sliding left → `[8,4,0,0]`. |
| All 16 cells filled but merges possible | Game continues. Only when no merges exist AND board is full does the game end. |
| Rapid repeated swipes | Queue input or lock input during animation (~150ms debounce). |
| localStorage unavailable (private browsing) | Catch exceptions silently; game plays without persistence. |
| NaN / corrupted save data | Validate on load; fall back to new game. |
