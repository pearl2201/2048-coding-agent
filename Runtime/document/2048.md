# Game Design Document: 2048

## 1. Project Overview
- **Project Name:** 2048
- **Type:** Puzzle / Number-merging casual game
- **Core Functionality:** Slide numbered tiles on a 4x4 grid to combine matching numbers and reach 2048
- **Target Users:** Casual gamers, puzzle enthusiasts

## 2. Visual Specification

### Scene Setup
- **Layout:** Centered 4x4 grid with rounded tile containers
- **Background:** Warm cream/beige gradient
- **Typography:** Bold sans-serif font (e.g., Montserrat, Roboto)

### Color Palette
- **Background:** #faf8ef (warm cream)
- **Grid Background:** #bbada0 (muted brown)
- **Empty Cell:** #cdc1b4 (light tan)
- **Tiles by Value:**
  - 2: #eee4da
  - 4: #ede0c8
  - 8: #f2b179 (orange)
  - 16: #f59563
  - 32: #f67c5f
  - 64: #f65e3b
  - 128: #edcf72 (yellow)
  - 256: #edcc61
  - 512: #edc850
  - 1024: #edc53f
  - 2048: #edc22e (gold, with glow effect)

### Animations
- **Tile Movement:** Smooth 150ms ease-out slide
- **Tile Merge:** Scale pop effect (1.0 → 1.15 → 1.0) over 100ms
- **New Tile Spawn:** Fade in + scale from 0.5 to 1.0 over 200ms

## 3. Game Mechanics

### Core Rules
- Player uses arrow keys (or WASD) to slide all tiles in one direction
- Tiles slide until they hit the edge or another tile
- Two tiles with the same value merge into one tile with doubled value
- After each move, one new tile (90% chance: 2, 10% chance: 4) spawns randomly
- Game ends when no valid moves remain


### Scoring
- Score increases by the value of each merged tile
- Best score is persisted to localStorage

### Win/Lose Conditions
- **Win:** Creating a 2048 tile (player can continue playing)
- **Lose:** Grid is full AND no adjacent tiles can merge

## 4. Interaction Specification

### Controls
- **Arrow Keys:** Slide tiles up/down/left/right
- **WASD Keys:** Alternative directional controls
- **New Game Button:** Reset grid and score

### UI Elements
- Score display with animated counter
- Best score display
- New Game button
- Game Over overlay (semi-transparent)
- Win overlay (shown once when 2048 is reached)

## 5. Technical Implementation

### Technology
- Single HTML file with embedded CSS and JavaScript
- No external dependencies required
- Responsive design (works on mobile)

### Data Structures
- 4x4 grid array storing tile values
- Each tile tracks: position, value, merged state, newly spawned state

### Performance
- Efficient move validation
- CSS transforms for hardware-accelerated animations
- Debounced input handling to prevent rapid-fire moves

## 6. Audio (Optional Enhancement)
- Soft click sound on tile movement
- Satisfying "pop" sound on tile merge
- Victory chime when reaching 2048

## 7. Acceptance Criteria

1. Grid initializes with two random tiles (both 2s or one 2 and one 4)
2. Arrow keys slide all tiles in correct direction
3. Matching tiles merge correctly and only once per move
4. New tile spawns after each valid move
5. Score updates correctly on merge
6. Best score persists across sessions
7. Game Over detected when no moves available
8. Win condition triggers overlay on reaching 2048
9. New Game button resets everything
10. All animations are smooth and performant
